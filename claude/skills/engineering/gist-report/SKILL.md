# Gist Report

Create a shareable styled HTML report via GitHub Gist. Returns a clickable preview link.

## Fast Path (use this)

Write ONLY the body HTML and pipe it to the helper script. **No doctype, head, or CSS needed** — the script wraps it in a dark-themed template with all styling pre-baked.

```bash
cat <<'BODY' | bash .claude/skills/gist-report/scripts/gist-report.sh FILENAME.html "TITLE"
<h1>Title Here</h1>
<p class="meta">Subtitle or attribution</p>

<h2>Section</h2>
<p>Content goes here.</p>

<div class="card card-green">
  <div class="card-title"><span class="num">1</span> Item Title</div>
  <p>Description</p>
</div>
BODY
```

The script outputs a single preview URL. Done.

## Available CSS Classes

The template includes pre-built components. Just use these in your body HTML:

| Class | Use For |
|-------|---------|
| `.meta` | Subtitle, author, date (muted text) |
| `.stats` + `.stat` | Flex row of metric badges |
| `.card` | Bordered content card |
| `.card-red` | Card with red left border (errors, failures) |
| `.card-green` | Card with green left border (fixes, success) |
| `.card-blue` | Card with blue left border (info, features) |
| `.card-yellow` | Card with yellow left border (warnings) |
| `.card-title` | Bold heading inside a card |
| `.num` | Numbered circle badge (e.g. `<span class="num">1</span>`) |
| `.callout` | Highlighted box with accent border (key takeaways) |
| `.label` | Small uppercase section label |
| `.footer` | Centered muted footer |
| `<pre><code>` | Code blocks (monospace, bordered) |
| `<blockquote>` | Indented quote with left border |
| `<table>` | Styled table with borders and header |

## Body HTML Patterns

### Stats row
```html
<div class="stats">
  <span class="stat"><strong>42</strong> likes</span>
  <span class="stat"><strong>7</strong> retweets</span>
</div>
```

### Numbered card list
```html
<div class="card card-green">
  <div class="card-title"><span class="num">1</span> First Item</div>
  <p>Description here</p>
</div>
```

### Key takeaway
```html
<div class="callout">
  <div class="label">Key Insight</div>
  <p>The important conclusion goes here.</p>
</div>
```

### Footer
```html
<div class="footer">Source: <a href="URL">link text</a> — Generated by Claude Code</div>
```

## Transcript Mode

Upload the current session transcript as a GitHub Gist (JSON format):

```bash
uvx claude-code-transcripts json --gist "$(ls -t ~/.claude/projects/$(pwd | tr '/' '-')/*.jsonl | head -1)"
```

Returns a single gist URL. The gist contains the full conversation — user messages, assistant responses, tool calls, and timestamps.

**Before sharing**: the transcript includes all tool output as-is; review it if the session touched credentials or sensitive data.

| Issue | Fix |
|-------|-----|
| "No such file" | No session files found — ensure you're in a project directory |
| gh auth error | Run `gh auth login` to authenticate |
| Empty transcript | Session just started — need at least one exchange |

## Rules

1. **Write body HTML only** — never include `<!DOCTYPE>`, `<html>`, `<head>`, or `<style>` tags
2. **Use the pre-built classes** — don't write custom CSS
3. **Keep it concise** — summarize, don't reproduce entire sources
4. **One bash command** — pipe the heredoc to gist-report.sh, output the URL, done
5. **Filename** — use kebab-case, always end in `.html`

## Gotchas

- **`gh` not authenticated**: `gist-report.sh` calls `gh gist create` — run `gh auth login` if the command fails with an auth error
- **Heredoc delimiter collision**: if your body HTML contains the word `BODY` on a line by itself, the heredoc terminates early; use a different delimiter (e.g. `ENDBODY`)
- **`<!DOCTYPE>` or `<html>` in heredoc**: the script wraps the body in its own template — including these tags produces double-wrapped HTML with broken layout
- **Large output (>10MB)**: GitHub Gist has a soft limit; very large reports may be truncated silently — split into multiple gists or trim the content
- **`uvx` required for transcript mode**: `uvx claude-code-transcripts` needs uv installed; if unavailable, install via `curl -LsSf https://astral.sh/uv/install.sh | sh`
